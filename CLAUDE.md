# 하이클래스 7기 강의 핵심 정리

## 1주차: 전환 상위 1% 상세페이지 원칙 (112p)

### 상세페이지의 본질
- 상세페이지 = 사람을 꼬시는 것 = 마음을 열게 만드는 것
- 고객 = 주인공(HERO), 내 서비스 = 조연(도우미)
- 고객의 심리적 여정: 일상 → 문제 직면 → 심각성 인식 → 제품 발견 → 해결 가능성 인식 → 혜택 학습 → 구매 결정

### 대원칙 1: 읽는 사람의 인생이 어떻게 변할까?
- "드릴보다는 구멍" — 제품 스펙이 아니라 고객 인생의 변화를 보여줘라
- 매슬로우 5대 욕구 활용 (안전/소속/존중/자아실현)
- 카테고리별: 뷰티→Before/After, 테크→시간절약/가성비, 스포츠→새로운경험

### 대원칙 2: 무조건 쉽게
- 뇌는 체중 2%이지만 열량 20% 사용 → 어려운 글 = 뒤로가기
- 쉽게 쓰기 7원칙:
  1. 맞춤법 지키기
  2. 불필요한 말 제거 (~에 관하여, ~라고 생각합니다)
  3. 문장 짧게 (40자 기준, 한 문장에 하나의 아이디어)
  4. 구체적 단어 사용 ("아름다운 인테리어" → "햇빛이 쏟아지는 대리석 바닥의 거실")
  5. 능동태 사용
  6. 전문용어 줄이기 → 고객 언어로
  7. 소리 내어 읽어보기

### 스토리텔링: 바람 vs 햇님
- 바람(자기자랑): "강력한 세정력, 계면활성제..." → 뒤로가기
- 햇님(고객 변화): "세탁소 아저씨에게 미안하다고 말하세요" → 구매

### 인트로 공식 3가지
- 공감형: 타겟의 고충에 공감 ("밤새 아이 재우느라 지친 부모님들...")
- 불안형: 방치 시 위험 강조 ("이것 모르면 절대 성공할 수 없습니다")
- 도파민형: 사용 후 즐거운 결과 ("이 제품의 부작용: 너무 예뻐져서...")
- 업종별 추천: 고가보험→불안, 육아→공감, 혁신제품→도파민

### 인지편향 (시스템1 vs 시스템2)
- 구매 결정의 95%는 시스템1(감정/직감) 담당
- 프레이밍 효과: 객관적 사실보다 어떻게 표현하는지가 더 중요
- 예) "90% 생존율" vs "10% 사망률" → 같은 수술, 완전히 다른 반응

### 디자인 핵심 원칙
- 핵심 카피: 확실히 크고 굵게 (860px 기준 135pt)
- 색: 2~3개 제한, 하이라이트 1개
- 배경-텍스트 명암 대비 확실히

### 1주차 과제

## 2주차: AI를 이용한 기획 및 카피라이팅 (190p)

### P.A.S.T.O.R 프레임워크
- P (Problem): 타겟의 페인포인트 명확히 제시
- A (Amplify): 문제 미해결 시 결과 강조
- S (Story & Solution): 스토리와 함께 해결책 제시
- T (Testimonials & Proof): 신뢰 구축 증거
- O (Offer): 구체적 제안과 혜택
- R (Response): 명확한 행동 유도 (CTA)

### ESR 감정 설계 프레임
- E (Expectation disruption): 기대 파괴 (1~5점)
  - 메시지_E: 전달 내용 자체가 예상을 깨는가
  - 형식_E: 화자/말투/톤이 카테고리 광고 문법을 깨는가
  - E는 가능하면 앞에 배치 (스크롤 멈추게)
  - S없이 E만 높으면 "기발한데 남 얘기"
- S (Self-relevance): 자기 관련성 (1~5점)
  - 감정을 "만드는"게 아니라 이미 있는 경험을 "꺼내는" 것
  - 시간, 장소, 감각, 행동이 구체적이어야 5점
- R (Resolution connection): 해결 연결성 (1~5점)
  - R은 감정 연결이 아니라 문제 재정의 능력
  - 문제의 진짜 원인을 새롭게 짚고 → 오직 이 제품만 가능한 구조
- 공식: 감정 강도 = E×S / 전환 확률 = E×S×R

### 타겟 설정 & 페르소나
- "모든 사람을 대상 = 아무도 대상 아님"
- 페르소나 6요소: 기본정보/일상/목표동기/고충/의사결정요소/구매장벽
- "5번의 왜" 기법: 표면 욕구 → 5번 왜를 물어 숨겨진 욕구 발굴

### 고객 인식 4단계
1. 무인식: 문제도 모름 → "이런 문제가 있다"부터 시작
2. 문제 인식: 문제는 알지만 해결책 모름 → "해결책이 있다"
3. 해결책 비교: 비교 중 → "우리가 나은 이유"
4. 구매 직전: 마지막 결정 → "지금 사야 하는 이유"

### 인지편향 프레이밍 7가지
1. 내부집단 + 확증편향: "마케팅 담당자라면 이미 알고 계시겠지만..."
2. 부정성 편향: 손실 회피 자극 "이것 모르면 절대..."
3. 서사 전달형: 이야기로 전달 → 기억↑ 비판↓
4. 헤일로 효과: 권위/실적 초반 배치
5. 밴드웨건: 다수 선택 증거 "5,231명이 전환..."
6. 앵커링: 높은 숫자 먼저 → 가격 낮아 보이게
7. 희소성: 제한된 시간/수량 "월 5명만, 남은 2자리"

### AI 프롬프트 작성법 (Claude)
- 기본 6요소: 역할/맥락/목적/형식/제약/추가정보
- 주의사항: 감정적 표현 자제, 인사말 생략, 반복 피하기
- XML 태그 고급 프롬프트: 정보 계층 구조화, AI 이해도 향상, 복잡한 지시 가능

### ESR 카피 평가 프롬프트 (과제 자료)
- 종합 점수: 감정강도(E×S)/25 + 전환확률(E×S×R)/125 + 가독성/30
- 등급: S/A/B/C/D
- 비주얼 체크: 비주얼 리드형(사진이 E담당) vs 카피 리드형(텍스트가 E/S/R 전부)
- 카피 파형 분석, 인식 단계 판별, 진입 전략 적절성 평가

# 김비서 — 업무 대시보드 (PWA 앱)

## 프로젝트 개요
PT 트레이너/피트니스 사업가를 위한 **통합 업무 관리 및 매출 분석 대시보드**. PWA 방식으로 배포되어 스마트폰 홈화면에 추가하면 앱처럼 사용 가능.

## 주요 기능
- **할 일 관리**: 우선순위/담당자/완료도 추적
- **일정 관리**: 월간 스케줄 및 이벤트
- **매출 분석**: 실시간 매출 데이터 및 차트
- **경제 뉴스**: 금융/시장 트렌드 요약
- **날씨 정보**: 업무 계획을 위한 실시간 날씨
- **제로코치 링크**: 전자책(크몽) + 인스타그램 다이렉트 링크

## 기술 스택
- **Frontend**: HTML5, CSS3 (반응형), Vanilla JavaScript
- **차트**: Chart.js
- **PWA**: Service Worker (오프라인 캐싱), Manifest.json
- **배포**: Vercel
- **URL**: https://qkrckddud.vercel.app

## 파일 구조
```
index.html              # 메인 대시보드 (HTML+CSS+JS 통합)
manifest.json          # PWA 메타데이터 (앱 이름, 아이콘, 테마)
service-worker.js      # 오프라인 캐싱 및 네트워크 관리
icon-192x192.png       # 홈화면 아이콘 (작음)
icon-512x512.png       # 설치 알림 아이콘 (크음)
create-icons.html      # 브라우저 기반 아이콘 생성 도구
```

## PWA 설치 방법
1. **Chrome/Edge**: 주소창 옆 📥 버튼 또는 메뉴 → "앱으로 설치"
2. **Safari (iOS)**: 공유 → "홈 화면에 추가"
3. 설치 후 주소창 없이 앱처럼 실행 가능
